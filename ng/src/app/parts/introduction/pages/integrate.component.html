#1 Integrate Angular 11 inside DNN best practices

Adding an Angular-App to DNN is kind of simple, but doing it right requires quite a bit of thought. These are the parts we solved really well - which we recommend you use as inspiration. These aspects are solved by the _AngularApp.cshtml and the helper shared/_AngularIntegration.cshtml:

  • Ensure we have the correct base tag in the header to get perfect routing
  • Include a helper-JS which provides DNN module information to Angular (for WebApi calls which require more headers like the RequestVerificationToken)
  • Show edition selection for super-users
  • Show the app in live or staging edition so our customer can see a new version before it's live. It will also remember the chosen edition in the cookie.
  • Parse/import the important parts from the index.html which angular created automatically.
  • Special integration for developers - developers can run ng serve on their local system and see it in DNN. This makes development 10x faster.
