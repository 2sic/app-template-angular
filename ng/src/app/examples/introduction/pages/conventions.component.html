Conventions used in the code

We strive to create all our demos / tutorials / templates according to best practices - while keeping a balance to making it simple. Here are some important conventions we use, just to help you better navigate our code:

  1. We use a lot of observables as is best practices in Angular. Since it's helpful to know what variables are observable streams, we usually add a $ to the end of the variable name. For example, a stream giving you a team is called team$.
  2. We split the application into the intro/root part, and all the examples are in a separate module. This is great for demonstrating sub-modules, sub-routing and more. It also allows you to just copy this app, remove the examples module and build whatever you want.
