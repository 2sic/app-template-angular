How it works

Dnn APIs require certain headers in http-requests. The most commen ist a RequestVerificationToken which is also known as the Anti Forgery Token. Additional headers like ModuleId or TabID provide context so the backend knows what system you are talking to.

These headers are automatically added to your http-requests thanks to dnn-sxc-angular. There are two ways dnn-sxc-angular gets these values:

  1. using the attributes in the app-tag (faster, but harder to setup)
  2. auto-detecting it from the HTML-DOM (slower, but just works)
